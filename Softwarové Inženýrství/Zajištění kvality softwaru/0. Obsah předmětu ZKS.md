## Státnicové minimum

- Popsat a porovnat V-model a W-model a vysvětlit, proč W-model lépe podporuje shift-left testing.
- Vysvětlit static testing, jeho roli ve W-modelu a znát základní metody: requirements review, design review, code review, walkthrough, inspection, static code analysis a quality gates.
- Vysvětlit princip MBT: test basis → SUT model → generované test cases/scenarios → test execution.
- Porovnat MBT s manuálním návrhem testů z hlediska investice, údržby, coverage, duplicit a vhodnosti pro komplexní systémy.
- Uvést příklady modelů pro MBT: state machine, workflow graph, UML activity/state diagram, BPMN process model, control flow graph nebo API model.
- Vysvětlit vztah MBT a test automation: MBT hlavně generuje testy, test automation hlavně spouští testy a vyhodnocuje výsledky.
- Vysvětlit princip test automation, ROI a kdy se automatizace ekonomicky vyplatí.
- Znát test automation pyramid: hodně unit testů, méně integration/service testů a málo UI/E2E testů.
- Znát úrovně automatizace testů: unit, integration/service, API, UI/E2E, performance/security a CI/CD quality gates.
- Vysvětlit equivalence partitioning, validní/nevalidní třídy a rozdíl mezi technicky nevalidními a businessově nevalidními daty.
- Vysvětlit boundary value analysis a proč se chyby často objevují na hranicích tříd.
- Vysvětlit combinatorial explosion a proč exhaustive testing není praktický.
- Vysvětlit CIT, pairwise/2-way testing, N-way testing, uniform strength, mixed-strength coverage a constraints.
- Umět popsat, jak efektivně redukovat kombinace vstupů: EC, BVA, pairwise/N-way, constraints a mixed-strength podle rizika.
- Vysvětlit path-based testing jako generování testů z cest přes grafový model systému.
- Znát formální model systému `G = (N, E)` nebo `G = (D, A)` a význam uzlů, hran, startu a koncových uzlů.
- Znát coverage kritéria: node coverage, edge coverage, edge-pair coverage, prime-path coverage a all-paths coverage.
- Vysvětlit prime path jako maximální simple path, která není vlastní podcestou jiné simple path.
- Vysvětlit, jak se priorita workflow aktivit modeluje pomocí priorit/barvení uzlů nebo hran a jak ovlivňuje sílu coverage při generování testů.
