# Rastrová grafika

> Ke státnicím: Raster graphics. Line drawing algorithms: DDA, Bresenham Algorithm. Filling algorithms: Polygon filling, Raster seed-based algorithms. Clipping lines and polygons.

## Základy rastrové grafiky

**Rastrový obraz** = 2D mřížka pixelů (picture elements). Převod spojitého geometrického popisu do diskrétní formy pixelů se nazývá **digitalizace** (2D) nebo **rendering** (3D).

**Přímka** v matematice je nekonečně tenká; v rastrové grafice se stává posloupností sousedních pixelů — výsledkem vzorkování spojité funkce.

**Směrnice** přímky:

```
k = Δy / Δx = (y2 - y1) / (x2 - x1)
```

Směrnice určuje **řídící osu** (osu, po které se pohybuje o 1 pixel za krok). Pokud |k| ≤ 1, řídící osa je X; pokud |k| > 1, řídící osa je Y.

## Algoritmy pro kreslení přímek

### 2.1 DDA — Digital Differential Analyzer (float)

**Princip:** Přímo využívat směrnici `k` (reálné číslo) k výpočtu dalšího pixelu — v každém kroku se k aktuální souřadnici přičítá `k`.

**Předzpracování:**

1. Zaokrouhlení souřadnic koncových bodů na celá čísla.
2. Určení řídící osy (|Δx| ≥ |Δy| → krok po X; jinak krok po Y).
3. Orientace zleva doprava (nebo zdola nahoru), aby krok byl vždy +1.

**Algoritmus (|k| ≤ 1, zleva doprava):**

```
k = (y2 - y1) / (x2 - x1)
y = y1
pro x = x1 až x2:
    setPixel(x, round(y))
    y += k
```

**Nevýhoda:** Práce s čísly s plovoucí desetinnou čárkou — pomalé, chyby zaokrouhlování se hromadí. Nahrazen Bresenhamovým algoritmem.

---

### 2.2 Bresenhamův algoritmus pro úsečku (pouze celá čísla)

> Vyvinul Jack Elton Bresenham v IBM, 1962; publikováno 1965.

**Princip:** Zcela eliminovat plovoucí desetinnou čárku. V každém kroku se rozhoduje, zda inkrementovat Y o 1 nebo ponechat, a to pouze na základě aritmetiky celých čísel.

#### Odvození (oktant 1: 0 ≤ k ≤ 1, x1 < x2)

Po přesunu z $x_i$ na $x_{i+1} = x_i + 1$ leží skutečná hodnota y na přímce:

```
y_real = k·(x_i + 1) + b
```

Definujeme vzdálenosti ke dvěma kandidátním pixelům:

```
d1 = y_real - y_i          (vzdálenost ke spodnímu pixelu)
d2 = (y_i + 1) - y_real   (vzdálenost k hornímu pixelu)
```

Rozhodnutí: vybereme pixel bližší skutečné přímce, tj. rozhodujeme podle znaménka `d1 - d2`.

Aby se **eliminovaly všechny zlomky**, násobíme `2Δx`:

```
p_i = 2Δx · (d1 - d2) = 2Δy·x_i - 2Δx·y_i + konst.
```

Toto je **rozhodovací proměnná** `p`. Její inkrementální aktualizace:

```
p_{i+1} = p_i + 2Δy - 2Δx·(y_{i+1} - y_i)
```

**Počáteční hodnota:**

```
p_0 = 2Δy - Δx
```

**Dvě větve:**

|Podmínka|Akce|Aktualizace p|
|---|---|---|
|`p < 0`|y zůstává (spodní pixel)|`p += 2Δy`|
|`p ≥ 0`|y se zvýší o 1 (horní pixel)|`p += 2Δy - 2Δx`|

**Implementace (pouze oktant 1):**

```c
void bresenham(int x1, int y1, int x2, int y2) {
    int dx = x2 - x1, dy = y2 - y1;
    int c0 = 2 * dy;          // přírůstek pokud p < 0
    int c1 = c0 - 2 * dx;    // přírůstek pokud p >= 0
    int p  = c0 - dx;        // p_0 = 2Δy - Δx
    setPixel(x1, y1);
    for (int x = x1 + 1; x <= x2; x++) {
        if (p < 0) {
            p += c0;
        } else {
            p += c1;
            y1++;
        }
        setPixel(x, y1);
    }
}
```

**V těle cyklu pouze sčítání a odčítání — žádná plovoucí desetinná čárka, žádné násobení.**

#### Zobecnění na všechny oktanty

Algoritmus v uvedené podobě funguje pouze pro oktant 1 (0 ≤ k ≤ 1). Pro ostatní oktanty:

- Prohodíme role X a Y, pokud |k| > 1.
- Negujeme přírůstky pro záporné Δx nebo Δy.
- Využijeme **8násobnou symetrii** (u kružnice) nebo podmíněné přepínání znamének (u úseček).

#### Bresenhamův algoritmus pro kružnici

Stejná myšlenka — místo odmocnin z rovnice kružnice se pracuje se čtverci (celá čísla):

```
p_1 = 3 - 2r
```

Větve:

```
pokud p < 0:  p += 4·x + 6
jinak:        p += 4·(x - y) + 10; y--
```

Využití **8násobné symetrie**: vypočítáme 1/8 kružnice, každý krok nakreslíme 8 symetrických pixelů pomocí `setSymPixel(x, y, cx, cy)`.

## Algoritmy vyplňování polygonů

### 3.1 Scan-line filling (vyplňování po řádcích)

**Princip:** Pro každý vodorovný řádek (scan-line) najdeme průsečíky s hranami polygonu, seřadíme je a vyplníme úseky mezi sousedními páry.

#### Předzpracování (před hlavním cyklem):

1. **Orientace hran** — první bod nahoře, druhý dole.
2. **Odstranění všech horizontálních hran** — způsobují dvojité počítání vrcholů.
3. **Systematické zkrácení každé hrany o 1 v y-souřadnici (zespoda)** — zabraňuje dvojímu vyplnění na společných vrcholech (pravidlo vlastnictví: levá a horní hrana patří oblasti).
4. Nalezení `YMAX` a `YMIN`.

#### Hlavní cyklus:

```c
for (int y = YMAX; y >= YMIN; y--) {
    najdi průsečíky scan-line y s aktivními hranami   // topologie
    seřaď průsečíky podle X                           // bubble sort (skoro seřazený seznam)
    vykresli horizontální úseky mezi sousedními páry
}
```

**Active list má vždy sudý počet uzlů** (každý úsek má začátek i konec).

#### Datová struktura hrany (6 položek):

|Položka|Typ|Mění se?|
|---|---|---|
|`Y_upper`|int|Ne|
|`Y_lower`|int|Ne|
|`step_X` (= Δx/Δy)|float|Ne|
|`X_upper`|float|Ano — přičítá se `step_X` každý řádek|
|`intersection_X`|int|Ano — `round(X_upper)`|
|`*next`|ukazatel|—|

#### Varianty:

**Inverzní plnění (XOR):** Není potřeba řazení. Pro každou hranu se provede XOR horizontální čáry od průsečíku do X_max. Vyplněná oblast nepřekrývá dříve nakreslené objekty.

**Stencil buffer:**

- _Nekonvexní_ oblasti: 1-bitový buffer, aplikujeme XOR operace, pak zkopírujeme do framebufferu.
- _Konvexní_ oblasti: pole dvojic celých čísel `[XL, XR]`, přímé vyplnění zleva doprava.

### 3.2 Vyplňování trojúhelníku — Juan Pineda (1988)

Trojúhelník je vždy konvexní. Využívá **hranovou funkci**:

```
E(P) = (xB - xA)·(yP - yA) - (yB - yA)·(xP - xA)
```

- `E(P) > 0` → P leží vlevo od hrany AB (uvnitř pro CCW orientaci)
- Pixel je uvnitř trojúhelníku, pokud: `E1(P) > 0 AND E2(P) > 0 AND E3(P) > 0`

Hranové funkce se aktualizují **inkrementálně** (jedno sčítání za krok) — velmi rychlé, přátelské pro GPU.

**Pravidlo vlastnictví hrany (top-left rule):** Pixel přesně na sdílené hraně patří trojúhelníku, jehož hrana je horní (top) nebo levá (left). Zajišťuje, že žádný pixel není nakreslen dvakrát při renderování sousedních trojúhelníků.

### 3.3 Seed-based algoritmy (vyplňování ze semínka)

**Princip:** Začínáme od označeného pixelu uvnitř oblasti a šíříme se do sousedních pixelů, dokud nenarazíme na hranici.

#### 4-spojité vs. 8-spojité sousedství

```
4-spojité:     [ ]
             [x][X][x]   (pouze nahoru/dolů/vlevo/vpravo)
               [ ]

8-spojité:   [x][x][x]
             [x][X][x]   (včetně diagonál)
             [x][x][x]
```

**Pozor na neshodu:** Pokud je hranice 4-spojitá, použij 8-spojité vyplňování (a naopak), jinak může výplň „vytéct" diagonálními mezerami.

#### Rekurzivní seed-fill

```python
def seed_fill(x, y, barva_hranice, barva_výplně):
    if pixel(x, y) == barva_hranice or pixel(x, y) == barva_výplně:
        return
    setPixel(x, y, barva_výplně)
    seed_fill(x+1, y, ...)
    seed_fill(x-1, y, ...)
    seed_fill(x, y+1, ...)
    seed_fill(x, y-1, ...)
```

**Problém:** Přetečení zásobníku pro velké oblasti (jedno rekurzivní volání na každý vnitřní pixel).

#### Scan-line seed-fill (zásobník segmentů)

Efektivnější: na zásobník se ukládají celé horizontální segmenty, ne jednotlivé pixely.

```
vlož (seed_x, seed_y) na zásobník
opakuj:
    vyndej semínko ze zásobníku
    najdi levý a pravý okraj → vyplň horizontální segment
    analyzuj řádek nad: pro každý volný podsegment vlož 1 semínko
    analyzuj řádek pod: pro každý volný podsegment vlož 1 semínko
dokud zásobník není prázdný
```

Hloubka zásobníku odpovídá počtu horizontálních segmentů, nikoli jednotlivých pixelů — výrazně efektivnější.

## Ořezávání (Clipping)

Osově zarovnané obdélníkové **ořezávací okno** je definováno jako `[xmin, xmax] × [ymin, ymax]`.

### 4.1 Cohen–Sutherlandovo ořezávání úseček (1967)

**Princip:** Každému koncovému bodu se přiřadí 4-bitový **outcode** označující, na které straně (stranách) okna leží.

```
Rozložení bitů: [NAHOŘE | DOLE | VPRAVO | VLEVO]
                [  8    |  4   |   2    |   1  ]
```

**Tři případy:**

|Podmínka|Význam|Akce|
|---|---|---|
|`OC1 OR OC2 == 0000`|Oba body uvnitř|Přijmout a nakreslit|
|`OC1 AND OC2 != 0000`|Oba body vně stejné hrany|Triviálně odmítnout|
|Jinak|Jeden nebo oba částečně vně|Oříznout o vnější hranu, přepočítat OC, opakovat|

**Výhoda:** Rychlé triviální přijetí/odmítnutí pro většinu úseček (málo skutečných výpočtů průsečíků).  
**Nevýhoda:** Pro úsečky procházející více hranami je potřeba více iterací.

---

### 4.2 Liang–Barskyho ořezávání úseček (1983/1984)

> „A New Concept and Method for Line Clipping", ACM TOG, Vol. 3, č. 1, 1984.

**Princip:** Parametrické vyjádření úsečky, jediný průchod přes všechny čtyři hranice.

```
x = x1 + t·Δx
y = y1 + t·Δy       kde t ∈ [0, 1]
```

Pro každou ze 4 ořezávacích hranic definujeme `p_k` a `q_k`:

|Hranice|p_k|q_k|
|---|---|---|
|Levá|`-Δx`|`x1 - xmin`|
|Pravá|`+Δx`|`xmax - x1`|
|Spodní|`-Δy`|`y1 - ymin`|
|Horní|`+Δy`|`ymax - y1`|

Podmínka `p_k · t ≤ q_k` musí platit, aby úsečka ležela uvnitř hranice k.

**Logika testovací funkce** (pro každou hranici):

```
t = q_k / p_k

pokud p_k < 0:              // úsečka vstupuje oknem touto hranicí
    pokud t > t2: odmítnout
    jinak pokud t > t1: t1 = t   // aktualizace vstupního parametru

pokud p_k > 0:              // úsečka vystupuje oknem touto hranicí
    pokud t < t1: odmítnout
    jinak pokud t < t2: t2 = t   // aktualizace výstupního parametru

pokud p_k == 0 a q_k < 0: odmítnout  // rovnoběžná a vně
```

Po průchodu všemi 4 hranicemi:

- `t1 > t2` → úsečka je celá vně, odmítnout.
- `t1 > 0` → vstupní bod byl oříznut (nový začátek = `P(t1)`).
- `t2 < 1` → výstupní bod byl oříznut (nový konec = `P(t2)`).

**Výhoda oproti Cohen–Sutherlandovi:** Jediný průchod, žádná iterace, přímo vypočítá oříznuté koncové body.

---

### 4.3 Sutherland–Hodgmanovo ořezávání polygonů (1974)

> „Reentrant Polygon Clipping", CACM Vol. 17, č. 1, 1974.

**Proč nestačí oříznout každou hranu zvlášť jako úsečku?** Nezávislé ořezávání jednotlivých hran polygonu by rozbilo uzavřenou hranici — chyběly by hrany a vznikaly by díry.

**Princip: Znovuzadávající pipeline (reentrant).** Polygon se ořezává postupně přes každou ze 4 hranic okna. Výstupní seznam vrcholů jednoho stádia jde přímo na vstup dalšího.

**Pro každou ořezávací hranu, 4 případy pro každou hranu polygonu (S → P):**

```
S = předchozí vrchol (již zpracovaný)
P = aktuální vrchol

Případ 1: S uvnitř,  P uvnitř  → výstup P
Případ 2: S uvnitř,  P vně     → výstup průsečík(S,P)
Případ 3: S vně,     P uvnitř  → výstup průsečík(S,P), pak P
Případ 4: S vně,     P vně     → žádný výstup
```

**Omezení:** Ořezávací okno musí být **konvexní**. Funguje pro libovolný konvexní polygon jako okno, nejen obdélník.

**Problém s nekonvexními vstupními polygony:** Fantomové hrany — nadbytečné hrany spojující dvě oddělené oříznuté části podél ořezávací hranice. Výstup je geometricky správný pro vyplňování, ale obsahuje extra hrany.

---

### 4.4 Weiler–Athertonovo ořezávání polygonů

Řeší **libovolné (i nekonvexní) ořezávací okno i nekonvexní vstupní polygon**.

**Datová struktura:** Dva obousměrné seznamy vrcholů — jeden pro vstupní polygon, druhý pro ořezávací polygon. Průsečíky jsou vloženy do obou seznamů s křížovými ukazateli.

**Algoritmus:**

1. Začít průchod od vstupního průsečíku (vstupní polygon vstupuje do ořezávacího polygonu).
2. Sledovat hranici vstupního polygonu.
3. Na průsečíku **přepnout na druhý seznam** (hranice ořezávacího polygonu).
4. Pokračovat, dokud se nevrátíme na začátek.
5. Každý průchod vytvoří jeden výstupní polygon.

**Kompromis:** Složitější implementace než Sutherland–Hodgman; nutný, pokud je ořezávací nebo vstupní polygon nekonvexní.

## 5. Srovnání algoritmů

### Kreslení přímek

|Vlastnost|DDA|Bresenham|
|---|---|---|
|Datový typ|float|pouze int|
|Operace|float sčítání, zaokrouhlení|int sčítání/odčítání|
|Rychlost|pomalejší|rychlejší|
|Přesnost|hromadí zaokrouhlovací chyby|přesné|

### Ořezávání úseček

|Vlastnost|Cohen–Sutherland|Liang–Barsky|
|---|---|---|
|Klíčová idea|Outcodes (4 bity)|Parametrické t|
|Průchody|Iterativní (více průchodů)|Jediný průchod|
|Nejlepší případ|Triviální přijetí/odmítnutí|Vždy jediný průchod|
|Nejhorší případ|Mnoho iterací|Vždy jediný průchod|

### Ořezávání polygonů

|Vlastnost|Sutherland–Hodgman|Weiler–Atherton|
|---|---|---|
|Ořezávací okno|Pouze konvexní|Libovolné|
|Vstupní polygon|Konvexní i nekonvexní|Konvexní i nekonvexní|
|Fantomové hrany|Ano (nekonvexní vstup)|Ne|
|Složitost implementace|Jednoduchá|Složitá|

## ⚠️ Časté státnicové chytáky

1. **Bresenhamova počáteční hodnota:** `p_0 = 2Δy - Δx`, nikoli `2Δy`. Vynechání `-Δx` je nejčastější chyba.
    
2. **Bresenhamovy větve (záleží na směru):**
    
    - `p < 0` → y zůstává → `p += 2Δy` (menší přírůstek)
    - `p ≥ 0` → y se zvýší o 1 → `p += 2Δy - 2Δx` (větší hodnota, může být záporná) Studenti tyto větve často zaměňují.
3. **Bresenham — pouze oktant 1:** Odvozený algoritmus funguje pro `0 ≤ k ≤ 1, x1 < x2`. Pro ostatní oktanty je nutné prohodit osy nebo negovat delty.
    
4. **Scan-line: odstranit VŠECHNY horizontální hrany**, nikoli jen spodní. Všechny horizontální hrany se odstraňují v předzpracování.
    
5. **Scan-line: systematické zkrácení hrany o 1 v y (zespoda)** — toto zabraňuje dvojímu vyplnění vrcholů. Bez toho se konvexní vrcholy vyplní dvakrát.
    
6. **Scan-line active list: vždy sudý počet uzlů.** Lichý počet signalizuje chybu ve zpracování hran.
    
7. **`intersection_X` vs. `X_upper`:** `X_upper` je průběžný float, ke kterému se každý řádek přičítá `step_X`. `intersection_X` je jeho zaokrouhlená celočíselná hodnota pro skutečné kreslení.
    
8. **Stencil buffer a konvexita:** Konvexní varianta (dvojice `[XL, XR]`) funguje jen proto, že konvexní oblast má na každém řádku právě jeden span. Pro nekonvexní polygony se musí použít XOR/1-bitová verze.
    
9. **Cohen–Sutherland — AND vs. OR:**
    
    - `OC1 OR OC2 == 0` → triviálně **uvnitř** (oba body nemají žádné vnější bity)
    - `OC1 AND OC2 ≠ 0` → triviálně **vně** (oba body sdílejí alespoň jeden vnější bit, stejná strana) Tyto podmínky se snadno pletou.
10. **Liang–Barsky — logika p < 0 vs. p > 0:**
    
    - `p < 0`: úsečka se pohybuje **směrem k** hranici → vstupní průsečík → aktualizovat `t1`
    - `p > 0`: úsečka se pohybuje **od** hranice → výstupní průsečík → aktualizovat `t2`
    - `p == 0, q < 0`: úsečka je rovnoběžná a leží vně → odmítnout celou úsečku
11. **Sutherland–Hodgman vyžaduje konvexní ořezávací okno.** Pokud je dotaz na ořezávání nekonvexním oknem, odpověď je Weiler–Atherton, nikoli Sutherland–Hodgman.
    
12. **Ořezávání polygonu ≠ opakované ořezávání čar.** Aplikace Cohen–Sutherland na každou hranu polygonu zvlášť je chybná — rozbije uzavřenou hranici.
